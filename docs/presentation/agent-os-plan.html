The idea

A standalone OS that grounds feature work

Agent OS is a standalone repo that talks to the Hub via MCP and carries a unified enterprise knowledge base. When anyone — human or AI agent — builds a new feature, the OS supplies the company guidelines, existing build docs, and tech stack as grounding. Build to the pattern, not from scratch.

Problem today
  • New features reinvent existing patterns
  • RBAC / Supabase conventions live in people's heads
  • No single place agents can consult before coding
What the KB gives
  • One queryable graph of "what exists" + "why/how"
  • Agents-first: MCP/API consumable, humans via Obsidian
  • Cheap to update; agents can append knowledge
Scope decomposition

Architecture · Approach 1

Single unified vault, one graph

One Obsidian vault holds two folders. graphify indexes the whole corpus into one graph. Curated notes [[link]] to generated node names, so re-indexing turns those links into real edges — binding why/how onto what-exists.

✓ Chosen — one graph
Simplest thing agents can query. graphify's incremental hashing solves regen cost. Curated + code co-exist and cross-link.
✕ Two graphs, federated
Cross-links break across boundaries; query layer gets complex. Bad for agents-first.
✕ Curated-only + Hub indexed apart
Two refresh paths, fragile name-based links.

Updated later

Three refresh commands + agent path

refresh:hub
Bump submodule, graphify update — AST-only, no LLM cost. For Hub code changes.
refresh:vault
Re-index after curated edits. graphify extract, incremental via manifest hashes.
refresh:all
Full rebuild from both layers. Budget-guarded.
Agent-driven (self-updating KB): an agent that learns something new writes a curated note from a template, then runs refresh:vault. Writes only under curated/ — never generated/.

graphify keeps a per-file hash manifest, so incremental re-index only re-extracts what changed.

Safety & guardrails

Fail loud, never lose data

  • Token-cost guard — counts corpus words, aborts over budget unless --force
  • Temp-swap — build to temp dir, swap in only on success; previous graph stays intact
  • Regression refuse — won't swap a graph with fewer nodes
  • Pre-commit hook — blocks any staged edit under vault/generated/
  • Dangling-link warn — lists unresolved [[links]], does not fail
  • Provenance — every refresh appends a line to .changelog.md
Each guard is a small pure function with its own unit test — the decision logic is verified without ever invoking graphify.

Definition of done

Smoke — refresh:all on a clean checkout yields a non-empty graph (nodes & edges > 0).
2
Binding — the add-a-desk playbook resolves its [[links]] to real nodes (no dangling).
3
Query acceptance — "how do I add a desk?" returns the desk playbook + desk community.
4
Guardrail — an edit under generated/ is rejected by the pre-commit hook.
Plus: all curated notes well-formed (frontmatter validation), and unit tests on every guard/helper.
